Attack with shorter keys (Kuehn 2003)

To generate a faked signature with the Kuehn method we first create the data block as we would for a regular PKCS#1 signature. The structure of the faked signature differs from the regular signature in the following points:
  • The padding string is shortened. Thus the data block is shifted further to the left in the signature. This is not recognized by faulty implementations of the signature validation algorithm. In the Kuehn attack the length of the padding string is a fixed value (46 bytes if the SHA-1 hash function is selected).
  • The unused space at the end of the signature is filled with the hexadecimal byte values 0xff.


Please notice that in order to carry out the attack you first have to create an RSA key pair and the data block in the corresponding mask.
